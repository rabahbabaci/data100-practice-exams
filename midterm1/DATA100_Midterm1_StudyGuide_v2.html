TOPIC 1

Pandas — Indexing & Operations

Lectures 2–4 Heaviest topic on exam

This is the most tested topic. You need to be able to trace code and predict outputs quickly.

The Core Indexing Rules

MethodSelects ByEnd Inclusive?
df['col']Column by name → Series—
df[['a','b']]Multiple columns → DataFrame—
.loc[row, col]Labels (names)Both endpoints ✓
.iloc[row, col]Integer positions (0-based)Excludes end ✗
🧠 Mental Model
.loc = looking up by name tag. .iloc = pointing with your finger counting 0, 1, 2…
If index labels are integers, .loc[1] finds label "1" while .iloc[1] finds position 1 — these can be different rows!

Exam Traps from FA25 Q1

df['a'][1] — This works! First gets column 'a' as a Series, then uses [1] which does label-based lookup on the Series index. If the index has label 1, returns that value. Answer: 6
df.loc['b', :][1] — Gets row with label 'b' as a Series (index = column names). Then [1] looks for label 1 in column names. If columns are strings like 'a','b','c', there's no label 1 → NA (error)
df.iloc[1, 2] — Row at position 1, column at position 2. If there are only 2 columns (positions 0 and 1), position 2 is out of bounds → NA (error)

Filtering & Chaining

# Boolean filtering
df[df['b'] == 7]                 # rows where b = 7
df[df['b'] == 7]['a'].iloc[0]  # get first 'a' value from those rows

# Multiple conditions — MUST use & | ~ with parentheses
df[(df['a'] > 3) & (df['b'] < 8)]
df[(condition1) | (condition2)]    # OR
df[~(condition)]                   # NOT

# Sorting + extracting
df.sort_values('a', ascending=False).head(1)['b'].iloc[0]
NaN trap (from SP25, SU25): df['col'] == np.nan does NOT work for finding NaN! NaN ≠ NaN in Python. Use df['col'].isna() or pd.isnull(df['col']) instead.

Merging

pd.merge(left, right, left_on='col1', right_on='col2')
# how='inner' (default) — only matching rows from both sides
# how='left'  — all from left + matching from right (NaN if no match)
# how='outer' — everything from both (NaN where missing)
Merge row count: For inner join, if key "Apple" appears 2× in left and 3× in right → 2 × 3 = 6 rows for that key. Total = sum across all keys.

Key Series/DataFrame Methods

MethodReturns
s.unique()Array of unique values
s.value_counts()Each unique value → count (descending)
s.idxmax()Index label of the max value
s.isnull() / s.isna()Boolean: True where NaN
s.fillna(val)Replace NaN with val
df.dropna()Remove rows with any NaN
s.isin([a, b])Boolean: True if in list
s.str.contains(pat)Boolean: string contains pattern
s.str.startswith(x)Boolean: string starts with x

🎯 Exam Strategy
Trace every chain step by step. Write what each intermediate result looks like (Series? DataFrame? scalar?). If ANY step causes an error → NA. The most common tricks: wrong index type, out-of-bounds iloc, NaN comparisons, and chaining after sort (index doesn't reset).

GroupBy Methods & Row Counts

MethodWhat it does# Rows out
.agg(func)One summary per group= # unique groups
.agg({col: func})Different func per column= # unique groups
.head(k)First k rows from each groupΣ min(k, group_size)
.filter(func)Keep entire groups if func returns TrueAll rows from passing groups

filter returns ORIGINAL rows — not a summary. If 2 groups (sizes 3 and 2) pass the filter, you get 3 + 2 = 5 rows of the original data back, not 2 summary rows.

Pivot Tables (from SP25 Q1k, SU25 Q1g)

df.pivot_table(
    values='score',       # what to aggregate
    index='category',     # rows
    columns='year',      # columns
    aggfunc='mean',      # how to aggregate (default='mean')
    fill_value=0         # replace NaN cells
)
Pivot table = groupby + reshape. It groups by the index and columns combinations, applies aggfunc to values, and spreads the result into a grid. fill_value fills cells where no data exists for that combination.

agg with dictionaries (SP25 Q1h)

# Different aggregation per column
df.groupby('role').agg({
    'oh_tickets': 'min',
    'ed_hours': 'max'
})
TOPIC 3

Data Cleaning & EDA

Lecture 5

Granularity (tested in SU25 Q1a, SP25 Q1b)

"What does each row represent?" Read the data description carefully. Example: if each row records a fruit's price at a specific store location, the granularity is "each unique combination of item and store address."

Variable Types (tested in SU25 Q1b)

TypeDescriptionExamples
QuantitativeNumbers you can do math onHeight, price, temperature, count
Qualitative nominalCategories with no orderColor, fruit name, city, chain name
Qualitative ordinalCategories with meaningful orderStar rating (1-5), ranking, education level
Tricky cases: Zip codes, phone numbers, jersey numbers → look like numbers but are nominal (you can't average them meaningfully). Rankings (1st, 2nd, 3rd) → ordinal (order matters but gaps between ranks aren't necessarily equal).

Data Quality Checks

• Impossible values (negative ages, 999°F temperatures)
• Missing data — NaN. Ask: is it missing randomly, or is there a pattern?
• Duplicates — df.duplicated()
• Inconsistent formats — "CA" vs "California"
• Outliers — genuinely extreme or data errors?

TOPIC 4

Regular Expressions

Lecture 6

Two question types: (1) Given a pattern, list the matches (FA25 Q3). (2) Write a pattern that extracts specific info (SP25 Q2, SU25 Q2).

Pattern Reference

PatternMatches
\d / \DDigit / Non-digit
\w / \WWord char (letter, digit, _) / Non-word
\s / \SWhitespace / Non-whitespace
.Any char except newline
* / + / ?0+ / 1+ / 0 or 1 of previous
{m} / {m,n}Exactly m / between m and n
[abc]ONE char: a, b, or c
[a-z] / [A-Z]Any lowercase / uppercase letter
[^abc]Any char NOT a, b, c
(x|y)String "x" OR string "y"
( )Capture group — findall returns group content only
^ / $Start / end of string
The #1 regex trap on DATA 100 exams:
[Homework|Lab] ≠ "Homework or Lab"
[ ] is a character class — matches ONE character from the set. This matches: H, o, m, e, w, r, k, |, L, a, or b.
For alternation: (Homework|Lab)
Capture group trap: When re.findall() encounters ( ), it returns only the captured group content, not the full match. If you have r"Data(\d+)" applied to "Data100", findall returns ['100'], not ['Data100'].

Writing Regex Patterns (SP25 Q2, SU25 Q2 style)

Example: Match course IDs after "Data" — either 3 digits, or 1-2 digits optionally followed by a letter from [c, s, x].

# Target: from "Data8, Data100, CS61A, Data88s" → ['8', '100', '88s']
pattern = r"Data(\d{3}|\d{1,2}[csx]?)"
# \d{3} → exactly 3 digits (matches "100")
# \d{1,2}[csx]? → 1-2 digits + optional letter (matches "8", "88s")
# Parentheses capture only the ID part, not "Data"

Python Regex Functions

FunctionReturns
re.findall(pat, s)List of all matches (or group contents if groups exist)
re.sub(pat, repl, s)String with all matches replaced
s.str.contains(pat)Boolean Series (for Pandas filtering)
s.str.extract(pat)DataFrame of captured groups (needs parentheses)
🎯 Exam Strategy
For "list the matches": Walk left to right through the string, character by character. At each position, try to match the pattern. When a match is found, note it and start the next search AFTER the match ends (no overlapping).

For "write a pattern": Break the problem into cases. Use | for alternation between cases. Use ( ) to capture only the part you want returned. Test mentally against each example string.

Reading Box Plots (SP25 Q1e)

A box plot shows 5 numbers: min, Q1 (25th percentile), median (50th), Q3 (75th), max. The "box" spans Q1 to Q3, the line inside is the median. Dots beyond the whiskers are outliers.

SP25 trap: "About 75% of TAs resolved more tickets than about 75% of Tutors" — this means the Q1 of TAs ≈ Q3 of Tutors. Read carefully: "75% of TAs did MORE than X" means X is the Q1 (25th percentile) of TAs.

KDE & Bandwidth

Lecture 8

KDE questions appear on almost every exam. They test whether you understand how bandwidth affects the curve.

How KDE Works

Place a kernel
(bump) at each point
→
Add all
bumps together
→
Normalize
so total area = 1

Bandwidth (α)

αEffectVisual
Too smallSpiky, noisy — each point is a sharp peakOver-fit to individual points
Just rightCaptures true distribution shapeSmooth but preserves real features
Too largeOver-smoothed — washes out featuresDensity "leaks" into empty regions, hides bimodality
SP25 Q3d: "Given α = 0.5, which plot is correct?" — α represents the total width of each kernel. Larger α = wider kernels = smoother. Compare kernel widths visually to pick the right option.
FA24 trap: "The distribution is unimodal" — if the histogram shows two clear peaks, it's bimodal. An over-smoothed KDE can hide the second peak, but the histogram tells the truth.

KDE Properties

• Total area under curve = 1 (it's a density)
• Each kernel has area 1/n
• Y-axis = density (CAN be > 1 — that's fine)
• Always ≥ 0 everywhere
• A valid kernel must itself integrate to 1 (if not, you need to multiply by a constant — SU25 Q3f)

🧠 Mental Model
Bandwidth is the "blur slider" on a photo. Too sharp = noisy details. Too blurry = lose important features. The KDE with the right bandwidth shows the true shape without the noise.
TOPIC 7

Sampling, Bias & Errors

Lecture 9

The Four Things That Can Go Wrong

TypeWhat it meansReduced by
Chance errorRandom variability — different samples give different resultsLarger sample size
Selection biasSampling method excludes part of the populationRandom sampling (SRS)
Response biasPeople answer dishonestly / influenced by wordingBetter question design, anonymity
Non-response biasNon-responders are systematically different from respondersFollow-ups, reminders
Bigger sample ≠ less bias. A larger sample reduces chance error (random variability) but does NOT fix selection, response, or non-response bias. Surveying 10,000 CS majors about campus food is still biased, no matter how big the sample.

Key Terms (tested in SP25 Q3, SU25 Q1h)

TermMeaning
PopulationThe entire group you want to learn about
Sampling frameThe list you actually draw names from
SampleThe people who actually end up in your study
SRSSimple Random Sample — every individual has equal chance
Convenience sampleSample whoever is easy → high selection bias
Stratified sampleDivide into groups (strata), SRS within each
Cluster sampleRandomly pick clusters, survey everyone in them
🧠 FA25 Q5 Walkthrough

Josh emails 100 SRS students, 62 respond. Jake suggests: email 200 SRS + send reminders.

For each change, ask: which category does this affect?

• 200 instead of 100 → more data → chance error: smaller

• Reminders → more people respond → non-response bias: smaller

• Both still use SRS from same list → selection bias: same

• Everyone answers truthfully → response bias: same

SP25 Q3b trap: "If Rachel gives BayPass to students who signed up voluntarily, can she avoid selection bias?" — No! Voluntary sign-up = convenience sample. Students who sign up may be systematically different (e.g., already use transit more). Only random selection avoids selection bias.
SU25 Q1h trap: "Does dropping corrupted data introduce selection bias?" — Only if the corruption was non-random (e.g., only affecting certain groups). If corruption was random, dropping those rows doesn't introduce bias.
🎯 Exam Strategy
For each proposed change: "Does this change WHO is sampled, HOW MANY, WHETHER people respond, or HOW they answer?" Only that category changes. Everything else stays the same. Don't overthink it — the answer is usually straightforward once you identify which category.

🧠 Quick-Fire Recall

Pandas: .loc = labels (inclusive), .iloc = positions (exclusive end). NaN == NaN is False — use .isna(). Single [] = Series, double [[]] = DataFrame.

GroupBy: .agg() reduces to 1 row/group. .filter() returns original full rows. .transform() = same shape. Pivot table = groupby + reshape.

Regex: [abc] = one char from set. (a|b) = alternation. ( ) with findall → returns group only. + = 1+, * = 0+, ? = optional.

Viz: 1 var distribution → histogram/KDE/box. 2 quant vars → scatter/hex/contour. Quant×Cat → side-by-side box/overlaid hist.

KDE: High α → over-smoothed, density leaks. Low α → spiky. Total area = 1.

Sampling: Bigger n → less chance error, NOT less bias. SRS minimizes selection bias. Reminders reduce non-response bias.
