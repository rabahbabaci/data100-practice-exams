Pattern 1: \d{2,4} — Matches 2 to 4 consecutive digits.

Scanning left to right: "TK-" then 0042 (4 digits ✓), "$" then 125 (3 digits ✓), "." then 00 (2 digits ✓), then 09 ✓, 12 ✓, 2025 ✓, then 45 ✓. But re.findall returns non-overlapping matches left to right, and {2,4} is greedy.

(b) [2.5 pts]

Staff want to extract ticket IDs from a log file. Each ticket ID has the format "TK-" followed by exactly 4 digits. Given:

log = "Sold TK-0042 and TK-1399, refunded TK-0007. Invalid: TK-55, TKX-9999"

The desired output is ['0042', '1399', '0007']. Which patterns produce this output? Select all that apply.

Show Solution

Answer: A, C, D

A ✓: The capture group (\d{4}) grabs exactly 4 digits after "TK-". re.findall returns only the group content → ['0042', '1399', '0007']. "TK-55" has only 2 digits so no match. "TKX-9999" doesn't start with "TK-".

🪤 Trap: The capture group () behavior of re.findall is the most commonly tested regex concept. With groups → returns group content only. Without groups → returns full match.

Question 6: KDE Computation 4 Pts

A music reviewer rates 5 albums on a scale of 0–100. The scores are: {25, 40, 40, 60, 85}. She uses a boxcar (uniform) kernel with bandwidth α = 30:

K_α(x, x_i) = 1/α,  if |x - x_i| ≤ α/2
              0,      otherwise
(a) [2.5 pts]

Compute the KDE density estimate at x = 45. Show your work.

Show Solution

For each data point, check if |45 − x_i| ≤ α/2 = 15:

• x₁ = 25: |45 − 25| = 20 > 15 → K = 0

• x₂ = 40: |45 − 40| = 5 ≤ 15 → K = 1/30

• x₃ = 40: |45 − 40| = 5 ≤ 15 → K = 1/30

• x₄ = 60: |45 − 60| = 15 ≤ 15 → K = 1/30

• x₅ = 85: |45 − 85| = 40 > 15 → K = 0

KDE = (1/n) × Σ K = (1/5)(0 + 1/30 + 1/30 + 1/30 + 0) = (1/5)(3/30) = (1/5)(1/10) = 1/50 = 0.02

Answer: 1/50 = 0.02

🪤 Trap 1: x₄ = 60 is exactly at the boundary (|45 − 60| = 15 = α/2). The condition is ≤ (less than or equal), so it IS included. Missing this gives 2/150 = 1/75 instead of 1/50.

🪤 Trap 2: The two data points at x = 40 each contribute their own kernel. Treating them as one point gives 1/75 instead of 1/50.

🪤 Trap 3: Don't forget to divide by n = 5! The raw kernel sum is 3/30 = 1/10, but the KDE formula includes the (1/n) factor.

(b) [1.5 pts]

Select True or False for each statement about KDE:

Increasing the bandwidth α produces a smoother KDE curve.
A valid KDE curve must have density values between 0 and 1 everywhere.
If we double n (the number of data points) while keeping α fixed, the height of each individual kernel contribution decreases.
Show Solution

Larger α → smoother: True ✓

A wider bandwidth spreads each kernel over a larger range, smoothing out local bumps. This is the fundamental bandwidth tradeoff: large α = oversmoothing, small α = undersmoothing. Tested on Fa24 Q3e.

Density must be between 0 and 1: False ✗

A KDE is a probability density function. The total area under the curve must equal 1, but individual density values CAN exceed 1. Think of a very narrow kernel centered on clustered data — the peak can be tall as long as the total area is 1.

Doubling n decreases individual kernel height: True ✓

The KDE formula is f̂(x) = (1/n) Σ K_α(x, xᵢ). Each kernel has height 1/α, but the (1/n) factor means each kernel's contribution to the final estimate is (1/n)(1/α). Doubling n halves each kernel's contribution. The curve stays normalized (area = 1) because there are now twice as many kernels, each contributing half as much.

Question 7: Sampling & Bias 4.5 Pts

The Greek Theatre wants to estimate what proportion of all concert-goers would attend a jazz festival. There are 50,000 ticket holders in their database. They consider three approaches:

Approach A: Survey every 10th person entering the venue during a rock concert.

Approach B: Select 500 ticket holders uniformly at random from the full database and email them the survey. 210 respond.

Approach C: Divide all 50,000 ticket holders into 5 strata by the genre they most recently attended (rock, pop, jazz, hip-hop, classical). Randomly sample 100 from each stratum.

(a) [1 pt]

Name the type of sample for each approach:

Approach A:

Approach C:

Show Solution

Approach A: Convenience sample

There is no randomization — they're sampling whoever happens to walk in during a specific event. This is a textbook convenience sample.

Approach C: Stratified random sample

The population is divided into non-overlapping strata (genre preference), and a random sample is drawn from each. This matches the definition exactly, as tested in SP25 Q3c.

(b) [2 pts]

For each statement about Approach B, select True or False:

This approach has zero selection bias because the 500 people were chosen at random.
The fact that only 210 of 500 responded could introduce non-response bias.
If the sample size were increased from 500 to 5,000 (same random method, same response rate), the non-response bias would decrease.
Chance error would likely be smaller for Approach B if the sample size were increased from 500 to 5,000.
Show Solution

Zero selection bias: True ✓

The 500 people were selected uniformly at random from the full population (all 50,000 ticket holders). The sampling frame matches the population, so there is no selection bias in who was selected. The bias issues arise from who responds, not who is selected.

Non-response bias possible: True ✓

Only 210/500 = 42% responded. If the people who respond differ systematically from those who don't (e.g., jazz fans might be more likely to respond to a survey about a jazz festival), this introduces non-response bias. Same principle as Fa25 Q5.

Larger sample fixes non-response bias: False ✗

Non-response bias is a systematic issue — it depends on who responds, not how many respond. If the same 42% response rate holds, the same type of people are responding, and the bias persists. Increasing sample size reduces chance error, not systematic bias.

Larger sample reduces chance error: True ✓

Chance error (random variation) decreases with larger samples. This is the fundamental distinction: sample size reduces random error but does NOT fix systematic biases (selection, response, non-response).

🪤 Trap: The "bigger sample fixes bias" misconception is tested on almost every Data 100 exam. Sample size reduces chance error ONLY. Systematic biases require methodological changes, not more data.
